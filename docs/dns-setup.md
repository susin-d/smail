# DNS Setup Guide

## Overview

Each domain you add to MaaS requires DNS records for email delivery, authentication, and spam prevention. This guide walks through configuring all required records.

## Prerequisites

- A domain registered with a DNS provider (Cloudflare, Route53, Namecheap, etc.)
- Your VPS public IP address
- DKIM public key (generated automatically when OpenDKIM starts)

## Required DNS Records

For each domain (e.g., `example.com`), configure the following:

### 1. A Record — Mail Server

Points mail server hostname to your VPS IP.

| Type | Name              | Value         | TTL  |
|------|-------------------|---------------|------|
| A    | mail.example.com  | YOUR_VPS_IP   | 3600 |

### 2. MX Record — Mail Exchange

Tells other servers where to send email for your domain.

| Type | Name        | Value             | Priority | TTL  |
|------|-------------|-------------------|----------|------|
| MX   | example.com | mail.example.com  | 10       | 3600 |

### 3. SPF Record — Sender Policy Framework

Specifies which servers are authorized to send email for your domain.

| Type | Name        | Value                | TTL  |
|------|-------------|----------------------|------|
| TXT  | example.com | `v=spf1 mx a ~all`  | 3600 |

### 4. DKIM Record — DomainKeys Identified Mail

Allows recipients to verify email signatures. The public key is generated by OpenDKIM at startup.

To retrieve your DKIM key:

```bash
# SSH into your VPS
docker exec maas-opendkim cat /etc/opendkim/keys/example.com/mail.txt
```

| Type | Name                            | Value                              | TTL  |
|------|---------------------------------|------------------------------------|------|
| TXT  | mail._domainkey.example.com     | `v=DKIM1; k=rsa; p=YOUR_KEY_HERE` | 3600 |

> **Note:** The DKIM TXT value can be very long. Some DNS providers require splitting into multiple strings of 255 chars each.

### 5. DMARC Record — Domain-based Message Authentication

Tells receiving servers what to do with emails that fail SPF/DKIM checks.

| Type | Name                | Value                                                             | TTL  |
|------|---------------------|-------------------------------------------------------------------|------|
| TXT  | _dmarc.example.com  | `v=DMARC1; p=quarantine; rua=mailto:postmaster@example.com; fo=1` | 3600 |

### 6. Reverse DNS (PTR Record)

Configure via your VPS provider (not DNS registrar). Maps your IP back to `mail.example.com`.

This is critical for email deliverability — many servers reject mail from IPs without proper rDNS.

## Verification

After configuring DNS records, verify them:

```bash
# Check MX record
dig MX example.com

# Check SPF
dig TXT example.com

# Check DKIM
dig TXT mail._domainkey.example.com

# Check DMARC
dig TXT _dmarc.example.com

# Check rDNS
dig -x YOUR_VPS_IP
```

## DNS Propagation

DNS changes can take up to 48 hours to propagate globally, but typically complete within 1-4 hours.

## Adding More Domains

Repeat the process above for each new domain. The MaaS API will return the required records when you add a domain via the dashboard.

## Cloudflare Users

If using Cloudflare:
- Set the `mail.example.com` A record to **DNS only** (gray cloud), NOT proxied
- MX records cannot be proxied
- Email traffic must connect directly to your VPS IP
